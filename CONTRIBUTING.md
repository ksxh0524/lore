# Contributing to Lore

Thanks for your interest in contributing! This guide will help you get started.

## Development Setup

### Prerequisites
- Node.js ≥ 20
- pnpm ≥ 9

### Steps

```bash
# 1. Fork and clone
git clone https://github.com/your-username/lore.git
cd lore

# 2. Install dependencies
pnpm install

# 3. Start development
pnpm dev
```

This starts both the server (port 3952) and client (port 5173) in parallel.

## Project Structure

```
packages/
├── server/     # Backend: Fastify + World Engine + Agent Runtime
├── client/     # Frontend: React 19 + Vite + shadcn/ui
└── shared/     # Shared TypeScript types
```

## Architecture Overview

Lore is an **event-driven AI life simulation**. Key concepts:

- **World Engine**: Manages time, events, and agent lifecycle
- **Agent Pool**: Each agent runs independently with personality, memory, and behavior
- **Event System**: Events are generated by rule engine (cheap) or LLM (expensive, used at decision points)
- **Memory Engine**: Long-term memory with vector search (SQLite vec0)

See [Technical Design Document](./docs/tech-design.md) for full details.

## Coding Conventions

- TypeScript strict mode
- ESLint + Prettier (format before commit)
- Shared types go in `packages/shared/`
- Keep it simple — we use AI-assisted coding, don't over-architect

## Commit Messages

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `refactor:` Code refactoring
- `chore:` Maintenance

## Pull Request Process

1. Create a feature branch from `main`
2. Make your changes
3. Ensure `pnpm build` passes
4. Open a PR with a clear description
