# Lore 项目概述

> 最后更新：2026-04-08 | 版本 v0.02

---

## 一句话定义

**Lore 是一个开源的 AI 世界模拟器。** 用户进入一个由 AI 驱动的虚拟世界——里面每个角色都是独立的 AI Agent，有自己的工作、社交、感情线、人生轨迹。世界持续运行，角色自主决策，用户的介入会改变世界走向，但不改变世界的自主性。

## 核心理念

**不是聊天工具，是一个有自己运行规则的虚拟世界。**

- AI 角色不是等着你找她聊，她有自己的工作、社交、感情线
- 你不理她三天，她可能已经谈了个新男朋友
- 有人可能出车祸死了，有人可能升职加薪了
- **世界不围着你转，你是其中一员**
- 每个角色都有完整的、不受限制的人生——送外卖的也可能创业成功变成你的竞争对手

## Lore 不是什么

- 不是聊天机器人（虽然有聊天功能）
- 不是 RPG 游戏（没有固定剧本）
- 不是 AI 助手套壳（Agent 有自主意志，行为不受限制）
- 不是多 Agent 编排框架（不是一个技术框架，是一个产品/游戏）

## 世界初始化

Lore 有两种世界初始化模式：

### 历史模式

基于真实历史时间点创建世界。用户可以**魂穿**成某个历史人物。

- "我要穿越到 2020 年当乔布斯，看看他不死的话苹果会怎样"
- "我要穿越到康熙年间九子夺嫡，魂穿成八阿哥"
- "我穿越到 2017 年，就过普通人的生活"
- 初始参数基于真实历史，但用户进去之后历史就**分叉**了——你改变了历史，后面不可能再按照真实历史发展
- 历史模式需要大量真实数据（人物、事件、社会背景），依赖**社区贡献**预设包

### 随机模式

完全随机生成一个世界。用户设定基础参数：

- **年龄**：15 岁、30 岁、60 岁……
- **地点**：中国、美国、英国、甚至 2080 年的火星
- **大致背景**：学生、上班族、无业……
- 其他的（父母是谁、家庭背景、人际关系、周围的人）**系统自动生成**，用户不能设置
- 没有魂穿概念

**初始化由专门的初始化 Agent 负责**（见 [初始化系统](../world/initialization.md)）。

## 玩家模式

进入世界后，用户可以在两种操控方式间切换：

### 角色模式（默认）

用户以自己的角色身份存在于世界中，和其他 Agent 平等。

### 上帝模式

用户脱离角色，以观察者视角看世界。**核心能力是观察**——看到所有 Agent 的思考过程、每一步决策。也可以通过触发事件间接影响世界（地震、飓风等），但不能直接修改 Agent 的数值或强制控制行为。

详见 [玩家模式](./player-modes.md)。

## 交互模式

不做 2D 地图，不做角色在屏幕上走动。采用**事件驱动 + 聊天 + 虚拟平台**的交互模式。

### 事件卡片（核心交互）

世界事件以卡片形式弹出，是用户感知世界运行的主要方式。

- 每张卡片：描述文字 + 可选配图 + 选择按钮（或仅通知）
- 事件类型：日常、社交、感情、突发、世界（天灾）
- 用户可选择介入或忽略
- 不介入 = 事件按 Agent 自主决策推进
- 动画效果：framer-motion 卡片弹出、滑入

### 聊天

- 与任意 Agent 1v1 对话
- 流式输出，打字机效果
- 支持 Markdown 渲染
- 用户可以上传照片、视频
- Agent 可以调用生图模型发自拍、发图片
- 聊天内容影响 Agent 对用户的态度和关系值
- Agent 根据当前心情和人格调整语气
- Agent 可以拒绝用户、可以消失、可以主动撩用户

### 虚拟平台

世界内部有虚拟的 YouTube、TikTok、Twitter 等平台：

- Agent 们会去刷、会去发内容
- Agent 可以调用生图模型生成自拍照发出来
- 用户也可以上传自己的照片和视频，"投放"到虚拟平台看 Agent 们的反响
- 模拟播放量、点赞、评论数据
- 平台本身也可能是 Agent 创业创建的

详见 [虚拟平台系统](../world/platform.md)。

### Agent 状态面板

- 左侧列表显示所有 Agent 及其状态概览
- 核心属性：心情、健康、金钱、职业、感情状态
- 普通模式：用表情符号表示心情（"开心"、"平静"、"难过"）
- 上帝模式：显示完整数值（"心情 75/100"、"金钱 12500"）

### 时间线

- 世界事件的时间轴，可回溯查看历史
- 底部水平滚动，关键事件高亮标注
- 点击可展开查看事件详情

### Monitor 面板

- 统计所有 Agent 的思考过程和每一步行为
- 默认折叠，不刷屏
- 上帝模式下默认展开
- 记录内容：Agent 的思考、决策、LLM 调用、token 消耗、状态变化

## 架构概览

### Lore 底层在干什么

**没有魔法。** Lore 底层就是这些：

```
1. Fastify HTTP 服务器      --> 提供页面 + REST API
2. WebSocket                --> 前端和服务端实时通信
3. setInterval 定时器        --> 每隔 N 秒触发一个 tick
4. 初始化 Agent              --> 世界首次创建时生成所有角色的完整生平
5. World Agent              --> 负责天灾、宏观事件等世界层面的模拟
6. N 个 AgentRuntime         --> 每个都是内存中的 TypeScript 对象，全部由 LLM 驱动思考
7. LLMScheduler              --> 所有 LLM 调用统一排队，优先级 + 并发控制 + 超载丢弃
8. EventBus                  --> Agent 间通过事件通信
9. SQLite 数据库             --> 持久化
```

### 每个 tick 做的事

```
tick() {
  1. 世界时间 +1
  2. World Agent 决策（天灾、宏观事件，LLM 驱动）
  3. 生成日常事件（时间驱动的固定事件）
  4. 生成概率事件（随机偶遇等）
  5. 遍历 Agent：每个都要思考（LLM 驱动）
     -> 频率分级：高频（用户身边） > 中频（有关联） > 低频（远处）
     -> LLMScheduler 统一排队，按优先级执行
     -> 超载时适当丢弃/延后低优先级请求
     -> 思考结果更新状态、存记忆
  6. 重要事件推给前端
  7. 定期持久化（每 10 个 tick 写一次 SQLite）
}
```

## 开发路线

| Phase | 主题 | 目标 |
|-------|------|------|
| 1 | 能跑 | 单 Agent + 世界初始化 + 基础沙盒 + 聊天 + 基础经济 |
| 2 | 能记 | 记忆引擎 + 多 Agent + 关系网络 + 虚拟平台 |
| 3 | 能活 | 自主行为 + 推送 + 上帝模式 + World Agent |
| 4 | 能玩 | 历史模式 + 社区预设 + 高级沙盒 |
| 5 | 能看 | 多 Provider + PWA + npm 发布 |
| 6+ | 能火 | 社区生态 + 上万 Agent + 高级功能 |
